      Classes

      AppCanvas Represents drawable canvas used by the BOOSE application. Supports drawing shapes, lines, movement and text.
      ArrayVariableT Represents a typed array variable that implements IVariable.
      CircleCommand Represents a command that draws a circle on the application canvas.
      ClearCommand Represents a command that clears all drawings from the canvas.
      CommandBase Base class for all BOOSE commands. Provides common properties like canvas, debug console, and parameter handling.
      CommandFactory Factory class for creating ICommand instances based on command names.
      CommandRunner The CommandRunner class is responsible for parsing and executing commands that manipulate the canvas, variables, arrays, loops, conditionals, and methods in the BOOSE environment.
      ConditionalProcessor Processes conditional statements (if, else if, else, endif) for BOOSE scripts. Keeps track of nested conditional contexts to determine whether commands should execute.
      ConditionEvaluator Evaluates simple conditional expressions of the form "variable operator value" using integer comparisons. Supported operators: <, >, ==, !=, <=, >=.
      DebugConsole The DebugConsole class is responsible for displaying debug messages on a given panel. It allows logging messages, clearing the console, and managing the display of each message.
      DrawToCommand Represents a command that draws a line from the current canvas position to a specified set of coordinates.
      Form1 Form1 is the main user interface class for the drawing application. It handles the user input, drawing commands, and manages the canvas and debug console.
      HelpForm Displays a help window that shows the BOOSE command reference.
      IntVariable Represents an integer variable that inherits from VariableBaseT.
      LoopProcessor Handles the processing of loop constructs (for and while) in BOOSE programs. Maintains loop state and calculates jump lines for loop control flow.
      MethodProcessor Handles method definitions and method calls in BOOSE programs. Supports parameter passing, return values, and local variable scoping.
      MoveToCommand Represents a command that moves the canvas cursor to a specified position without drawing.
      PenCommand Represents a command that sets the current drawing colour of the canvas using RGB values.
      PenDashCommand Represents a command that sets the dash style of the canvas pen.
      PenWidthCommand Represents a command that sets the width of the canvas pen.
      RealVariable Represents a variable of type Double in the BOOSE system. Inherits common functionality from VariableBaseT.
      RectCommand Represents a command that draws a rectangle on the canvas using the specified width and height.
      ResetCommand Represents a command that resets the canvas, clearing all drawings and resetting the cursor position to (0, 0).
      StarCommand Represents a command that draws a star on the canvas at the current cursor position, given the number of points and radius.
      TriangleCommand Represents a command that draws a triangle on the canvas using the specified base and height.
      VariableBaseT Represents a generic variable in the BOOSE system. Provides common functionality for strongly-typed variables like IntVariable and RealVariable.
      VariableStore Singleton class that manages all variables in the BOOSE environment. Provides methods to add, retrieve, check existence, and snapshot/restore variables.
      WriteCommand Represents a command that writes text or evaluated expressions to the canvas at the current cursor position.

      Interfaces

      ICommand Represents a command that can be executed in the BOOSE system.
      IVariable Represents a generic variable with a name, type, and value.
